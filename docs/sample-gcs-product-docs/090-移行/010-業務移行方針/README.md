# 010-業務移行方針

業務のやり方をどのように新システムへ移行するかの方針を記述します。

## ドキュメントの目的

- 現行業務から新システムへの移行方針（移行対象、移行方法、移行スケジュール、リスク対策等）を明確化し、円滑な業務移行を実現する。
- 関係者間の認識合わせ、移行計画・実施の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 業務移行方針の概要
- 移行対象業務・データ一覧
- 移行方法・スケジュール・リスク対策
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 業務移行の方針を明確にすることで、移行計画・実施の品質と効率を高める。
- 関係者間の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 業務移行の方針が曖昧になり、移行計画・実施で誤解や手戻りが発生する。
- 移行作業の品質低下・トラブルにつながる。

## サンプル

### 業務移行方針（小売業務システム例/表形式）

| 移行対象業務      | 移行方法         | 移行スケジュール | リスク対策           | 備考           |
|-------------------|------------------|------------------|----------------------|----------------|
| 商品マスタ管理    | CSV一括取込      | 2025/12/01       | 事前データ検証       | 現行システムより抽出 |
| 顧客情報管理      | 手動移行         | 2025/12/02       | 二重登録防止         | 重要顧客のみ   |
