# 090-システム化機能一覧

業務イベントに対して、システムで実現する機能を一覧化します。

## ドキュメントの目的

- 業務イベントごとに必要なシステム機能を整理し、設計・開発・テスト・運用の品質を高める。
- 業務要件とシステム機能の対応関係を明確化する。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 業務イベントとシステム化機能の一覧表（イベント名、機能名、概要、関連オブジェクト等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 業務要件とシステム機能の対応関係を明確にすることで、設計・開発・テスト・運用の品質と効率を高める。
- 機能漏れや認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 業務要件とシステム機能の対応が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- 機能漏れや認識ズレによる品質低下・トラブルにつながる。

## サンプル

### 業務イベントとシステム化機能一覧（小売業務システム例）

| 業務イベント | システム化機能     | 機能概要                     | 関連オブジェクト |
|--------------|--------------------|------------------------------|------------------|
| 販売         | POSレジ会計        | 商品販売・会計処理            | 商品、顧客、在庫 |
| 仕入         | 仕入伝票登録       | 仕入情報の登録・在庫反映      | 商品、仕入先、在庫|
| 在庫調整     | 棚卸・在庫修正     | 在庫数の修正・履歴管理        | 商品、在庫        |
| 返品         | 返品受付・処理     | 返品受付・在庫戻し            | 商品、顧客、在庫 |
| 発注         | 発注伝票作成       | 発注情報の登録・管理          | 商品、仕入先      |

---

注：上記は例です。実プロジェクトの業務イベント・システム化機能を具体的に記載してください。
