# 060-画面仕様

業務ユーザー視点での画面の目的・表示項目・操作を記述します。

## ドキュメントの目的

- 業務ユーザーが利用する画面の目的・表示項目・操作・遷移を明確化し、設計・開発・テスト・運用の品質を高める。
- 画面設計・UI/UX改善・マニュアル作成時の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 画面ごとの目的・表示項目・操作・遷移
- 画面レイアウトやワイヤーフレーム（必要に応じて）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 画面仕様を明確にすることで、設計・開発・テスト・運用の品質と効率を高める。
- UI/UXの認識ズレや手戻り・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 画面仕様が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- UI/UXの認識ズレによる品質低下・トラブルにつながる。

## サンプル

### 画面仕様例

- 画面名：在庫一覧画面
  - 目的：店舗在庫の状況を一覧で把握し、発注判断を支援する
  - 表示項目：商品ID、商品名、在庫数、発注点、発注ボタン
  - 操作：在庫数の確認、発注ボタン押下で発注画面へ遷移

- 画面名：発注画面
  - 目的：不足商品の発注を行う
  - 表示項目：商品ID、商品名、発注数量、仕入先選択、発注確定ボタン
  - 操作：発注数量入力、仕入先選択、発注確定ボタン押下

---

注：上記は例です。実プロジェクトの画面仕様・表示項目・操作・遷移を具体的に記載してください。
