# 060-システムテスト

システムテストのテストケースや結果概要を記述します。

## ドキュメントの目的

- 業務システムのシステムテストケース・結果（業務シナリオ、入力値、期待結果、実施状況等）を明確化し、設計・開発・運用の品質を高める。
- テスト設計・実施・品質保証の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- システムテストケース一覧表（業務シナリオ、入力値、期待結果、実施状況等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- システムテストケース・結果を明確にすることで、設計・開発・運用の品質と効率を高める。
- テスト設計・実施・品質保証の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- システムテストケース・結果が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- テスト設計・実施・品質保証の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### システムテストケース（小売業務システム例/表形式）

| 業務シナリオ     | 入力値           | 期待結果               | 実施状況   | 備考                 |

| 販売～在庫～発注  | 商品コード=A123、数量=2 | 在庫数が2減算→発注点未満で発注候補 | ○         | 正常系               |
