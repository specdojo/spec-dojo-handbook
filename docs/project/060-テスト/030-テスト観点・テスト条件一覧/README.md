# 030-テスト観点・テスト条件一覧

テスト観点と代表的な条件を表形式で整理します。

## ドキュメントの目的

- 業務システムのテスト観点・テスト条件（機能、非機能、例外、境界値等）を明確化し、設計・開発・運用の品質を高める。
- テスト設計・実施・品質保証の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- テスト観点・テスト条件一覧表（観点、条件、期待結果、備考等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- テスト観点・テスト条件を明確にすることで、設計・開発・運用の品質と効率を高める。
- テスト設計・実施・品質保証の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- テスト観点・テスト条件が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- テスト設計・実施・品質保証の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### テスト観点・テスト条件一覧（小売業務システム例/表形式）

| 観点         | 条件                       | 期待結果                   | 備考                 |
|--------------|----------------------------|----------------------------|----------------------|
| 機能         | 正常系（在庫登録）         | 在庫数が正しく登録される     | 画面・API両方で確認   |
| 機能         | 異常系（在庫数マイナス）   | エラー表示・登録不可         | バリデーション必須    |
| 境界値       | 在庫数=0/発注点未満         | 欠品アラート表示             | 境界値テスト          |
| 非機能       | レスポンス1秒以内           | 主要画面で1秒以内に表示      | 負荷試験で検証        |
| 例外         | DB接続失敗                  | エラー表示・管理者通知        | 障害テスト            |

---

注：上記は例です。実プロジェクトのテスト観点・テスト条件を具体的に記載してください。
