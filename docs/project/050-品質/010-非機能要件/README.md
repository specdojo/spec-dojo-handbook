# 010-非機能要件

性能・可用性・操作性・セキュリティなどの非機能要件を記述します。

## ドキュメントの目的

- 業務システムの非機能要件（性能、可用性、操作性、セキュリティ、保守性等）を明確化し、設計・開発・運用の品質を高める。
- 非機能設計・運用・テストの根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 非機能要件一覧表（要件区分、内容、指標・基準、備考等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 非機能要件を明確にすることで、設計・開発・運用の品質と効率を高める。
- 非機能設計・運用・テストの認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 非機能要件が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- 非機能設計・運用・テストの認識ズレによる品質低下・トラブルにつながる。

## サンプル

### 非機能要件（小売業務システム例/表形式）

| 要件区分   | 内容                         | 指標・基準                | 備考                   |
|----------|------------------------------|--------------------------|------------------------|
| 性能      | レスポンス1秒以内             | 主要画面操作時            | 負荷試験で検証         |
| 可用性    | 月間稼働率99.9%以上           | システム全体              | 障害時は自動復旧        |
| 操作性    | 直感的なUI・操作ガイド表示     | 主要画面でガイド表示       | ユーザーテストで検証    |
| セキュリティ| ログイン認証・権限管理         | パスワード強度・権限設定   | 脆弱性診断を実施        |
| 保守性    | 設定変更は管理画面から可能      | 主要パラメータ変更         | 管理者向けマニュアル整備 |

---

注：上記は例です。実プロジェクトの非機能要件を具体的に記載してください。
