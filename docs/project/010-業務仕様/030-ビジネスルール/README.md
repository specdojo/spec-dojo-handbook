# 010-ビジネスルール

業務上の判断・処理ルールを日本語で記述し、必要に応じて疑似コード（日本語TypeScript）も記載します。

## ドキュメントの目的

- 業務で必要となる判断基準や処理ルールを明文化し、関係者間の認識ズレや属人化を防ぐ。
- システム設計・テスト・運用時の根拠となるルールを整理する。

## ドキュメントの内容

本READMEには、少なくとも以下を含めます。

- 業務ルールの一覧と概要
- 各ルールの目的・背景・適用範囲
- ルールの具体的な記述（日本語、必要に応じて疑似コード）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 業務ルールを明文化することで、設計・開発・テスト・運用の品質と効率を高める。
- ルールの属人化や認識ズレによる手戻り・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 業務ルールが曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- ルール変更時の影響範囲や根拠が不明瞭になり、品質低下やトラブルにつながる。

## サンプル

### 業務ルール例

- 商品の発注点（在庫が○個未満になったら自動発注）
- セール期間中は割引率を○％に設定
- 発注は毎週月曜午前中にまとめて実施

### 疑似コード例（日本語TypeScript）

```typescript
// 商品発注点ルール
if (在庫数 < 発注点) {
 自動発注する();
}

// セール割引ルール
if (セール期間中) {
 商品価格 = 商品価格 * (1 - 割引率/100);
}
```

注：上記は例です。実プロジェクトの業務ルール・疑似コードを具体的に記載してください。
