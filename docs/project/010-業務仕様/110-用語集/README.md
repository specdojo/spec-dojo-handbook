# 020-用語集

業務で使う用語の定義を YAML 形式で管理します。

## ドキュメントの目的

- 業務・システムで使う用語の意味・使い方を明確化し、認識ズレや誤解を防ぐ。
- 要件定義・設計・テスト・マニュアル作成時の共通言語として活用する。

## ドキュメントの内容

本READMEには、少なくとも以下を含めます。

- 用語の一覧と定義（YAML形式）
- 各用語の意味・使い方・関連語
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 用語の定義を明確にすることで、業務・システム間の認識ズレや誤解を防ぐ。
- 新規メンバーや外部関係者への説明・教育が容易になる。

## このドキュメントがないとどう困るか

- 用語の意味が曖昧になり、要件・設計・テスト・運用で誤解や手戻りが発生する。
- ドキュメントや会話で使われる言葉の定義が統一されず、品質低下やトラブルにつながる。

## サンプル

### 用語定義例（YAML形式）

```yaml
商品:
 定義: 店舗で販売する物品。SKU単位で管理。
 関連語: 在庫, 発注, 売上

在庫:
 定義: 店舗や倉庫に保管されている商品の数量。
 関連語: 商品, 発注, 欠品

発注:
 定義: 商品を仕入先に注文する業務。
 関連語: 商品, 在庫, 仕入先
```

---

注：上記は例です。実プロジェクトの用語・定義・関連語を具体的に記載してください。
