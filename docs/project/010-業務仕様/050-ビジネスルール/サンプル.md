# ビジネスルール（BR）

## BR-001: 発注点を下回った商品の発注候補化

- 条件  
  - 対象商品が「定番商品」である  
  - 現在在庫数 < 発注点
- 動作  
  - 「発注候補一覧」に自動的に掲載する

### 疑似コード（日本語 TypeScript）

```ts
type 商品 = {
  商品ID: string;
  名前: string;
  定番商品フラグ: boolean;
  発注点: number;
};

type 在庫 = {
  商品ID: string;
  現在在庫数: number;
};

function 発注候補判定(商品情報: 商品, 在庫情報: 在庫): boolean {
  if (!商品情報.定番商品フラグ) {
    return false; // 定番以外は自動発注対象外
  }
  return 在庫情報.現在在庫数 < 商品情報.発注点;
}
```

## BR-002: つけ販売の限度額

- 条件  
  - 顧客の「つけ残高」 + 今回購入金額 > 顧客ごとの「つけ限度額」
- 動作  
  - つけ販売を禁止し、現金払いを案内する
