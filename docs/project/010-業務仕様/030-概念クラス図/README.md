# 050-概念クラス図

業務上のエンティティ（商品、顧客、在庫など）の属性、エンティティ間関係の全体構造を、概念レベルのクラス図により可視化します。

## ドキュメントの目的

- 業務・システムで扱う主要エンティティとその関係性を可視化し、設計・開発・テスト・運用の共通認識を作る。
- データモデルの設計の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 業務・システムの主要エンティティ一覧
- エンティティ間の関係（関連・集約・継承など）

## ドキュメントの作成責任者

業務担当が主導し、開発担当と連携して作成します。

## ドキュメントの読者

プロジェクトに関わる全ての関係者（業務担当、開発担当、テスト担当、運用担当など）。

## ドキュメントの利用タイミング

プロジェクトの全てのフェーズ（要件定義、設計、開発、テスト、運用）で参照・更新します。
