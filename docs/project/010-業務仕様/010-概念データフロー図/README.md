# 040-概念データフロー図

業務領域の全体的な構造を、概念レベルのデータフローにより可視化する。

## ドキュメントの目的

- 業務（プロセス）で扱う情報や物の流れ、業務のトリガーとなるイベント、担当者を可視化し、対象業務領域の俯瞰的な理解を促進する。
- 要件定義・設計・テスト・運用時のデータ連携や処理順序の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 業務（プロセス）とその内容、担当者、業務が発生するイベント、インプット、アウトプットを記載した一覧表
- 一覧表を基にした概念データフロー図（Mermaid記法）

## ドキュメントの作成責任者

業務担当が主導し、開発担当と連携して作成します。

## ドキュメントの読者

プロジェクトに関わる全ての関係者（業務担当、開発担当、テスト担当、運用担当など）

## ドキュメントの利用タイミング

プロジェクトの全てのフェーズ（要件定義、設計、開発、テスト、運用）で参照・更新します。
