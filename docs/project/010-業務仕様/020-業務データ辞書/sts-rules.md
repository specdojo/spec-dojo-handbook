# ステータス（Status）作成ルール

業務上の主要エンティティが取り得る状態（ステータス）を一覧で整理します。
各ステータスの正式名称・通称・状態の説明・管理場所などを明記し、業務プロセスにおける状態遷移や管理単位を明確にします。

## 1. メタデータ

| 項目       | 説明                            | 必須 |
| ---------- | ------------------------------- | ---- |
| id         | ステータス一覧ID (sts-xxx-xxxx) | ○    |
| type       | `domain` 固定                   | ○    |
| title      | ステータス一覧名                | ○    |
| status     | `draft`/`ready`/`deprecated`    | ○    |
| version    | バージョン                      | 任意 |
| owners     | 担当者                          | 任意 |
| tags       | タグ・分類                      | 任意 |
| supersedes | 置き換え関係（古仕様→新仕様）   | 任意 |

### 1.1 ID規約

- ステータス一覧IDは `^sts-[a-z0-9-]+$` に合致する一意IDを用いる。
- `type: domain` は業務ドメイン仕様であることを示す。

## 2. 記述ルール

- エンティティのステータスを 項目説明 と 値の説明（ステータス一覧） の二つの表で整理する。
- ステータス（正式名称）＝用語集の term と一致させる。
- 呼称＝現場で使用される言い方（例：店主・おばあちゃんが日常使う表現） を記載する。
- ステータスの説明には 状態の意味＋成立条件（いつこの状態になるか）＋終了条件（次に進む条件） を簡潔に含める。
- 値（value）はシステム上の識別子として lower-kebab-case を使用する。
- 業務種類に応じて保管場所や管理帳簿などの追加列を任意で拡張してよい。
- 略語のみの記述や曖昧な説明は避ける。

### 2.1 項目説明

| 列名             | 説明                                               |
| ---------------- | -------------------------------------------------- |
| **エンティティ** | 状態が属する業務上の主要対象（例：お金、注文など） |
| **項目**         | 状態を表す項目（フィールド）名                     |
| **説明**         | 項目の意味・役割・業務上の扱い                     |

### 2.2 ステータス説明

| 列名                 | 説明                                                            |
| -------------------- | --------------------------------------------------------------- |
| **値（value）**      | システム上で扱うステータスコード。`lower-kebab-case` を用いる。 |
| **ステータス**       | 状態の正式名称（日本語単数形）。用語集 term と一致させる。      |
| **呼称**             | 現場で使われる日常的な言い方。                                  |
| **説明**             | 状態の意味・成立条件・終了条件を明快に記載する。                |
| **保管場所**（任意） | 対象が物理的/論理的に存在する場所。                             |
| **管理帳簿**（任意） | 状態を記録・管理する帳簿や台帳。                                |

### 2.3 詳細表の列順

以下の順に統一する：

値 →ステータス → 呼称 → 説明 → 保管場所（任意） → 管理帳簿（任意）

## 3. サンプル

### 3.1 メタデータ

```yaml
---
id: sts-money
type: domain
title: お金のステータス
version: 0.1.0
status: draft
owners: []
tags: []
supersedes: []
---
```

### 3.2 お金のステータス例

#### 項目説明

| エンティティ | 項目             | 説明                     |
| ------------ | ---------------- | ------------------------ |
| お金         | お金のステータス | お金の流れにおける各状態 |

#### ステータス説明

| 値                       | ステータス | 呼称             | 説明                                               | 保管場所 | 管理帳簿           |
| ------------------------ | ---------- | ---------------- | -------------------------------------------------- | -------- | ------------------ |
| sales-proceeds           | 受領直後金 | もらったお金     | お客様から受け取って、まだレジに入れたばかりのお金 | レジ     | レジ記録           |
| cash-in-register         | レジ現金   | レジのお金       | レジの中に入っている、おつりと売上金のまとまり     | レジ     | レジ記録           |
| change-fund              | 釣銭準備金 | おつり用のお金   | 朝レジに入れておく、おつり用のお金                 | レジ     | レジ記録           |
| cash-transferred-to-safe | 金庫保管金 | 金庫のお金       | レジ締め後に金庫へ移したお金                       | 金庫     | レジ記録、会計帳簿 |
| bank-deposit             | 銀行預入金 | 銀行に入れたお金 | 金庫から持って行って、銀行に預けたお金             | 銀行口座 | 会計帳簿           |
