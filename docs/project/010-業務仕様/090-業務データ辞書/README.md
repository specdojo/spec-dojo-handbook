# 030-業務データ辞書

業務で扱う「項目」の意味や単位などを、エンティティごとの YAML で定義します。

## ドキュメントの目的

- 業務・システムで扱うデータ項目の意味・型・単位・制約を明確化し、設計・開発・テスト・運用の品質を高める。
- データ連携や帳票設計、外部システム連携時の共通仕様として活用する。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- エンティティごとの項目一覧と定義（YAML形式）
- 各項目の意味・型・単位・制約
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- データ項目の定義を明確にすることで、設計・開発・テスト・運用の品質と効率を高める。
- データ連携や外部システム連携時の仕様不一致・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- データ項目の意味や型・単位が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- データ連携や帳票設計時に仕様不一致・品質低下・トラブルにつながる。

## サンプル

### データ定義例（YAML形式）

```yaml
商品:
 商品ID:
  型: string
  定義: 商品を一意に識別するID
 商品名:
  型: string
  定義: 商品の名称
 単価:
  型: number
  単位: 円
  定義: 商品1個あたりの販売価格

在庫:
 商品ID:
  型: string
  定義: 商品を一意に識別するID
 在庫数:
  型: number
  単位: 個
  定義: 店舗や倉庫に保管されている商品の数量
```

---

注：上記は例です。実プロジェクトのデータ項目・型・単位・定義を具体的に記載してください。
