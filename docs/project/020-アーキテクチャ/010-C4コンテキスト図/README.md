# 010-C4コンテキスト図

C4コンテキスト図に関するMermaid図などを配置します。

## ドキュメントの目的

- システム全体の構成要素（人・外部システム・主要サブシステム等）と関係性を俯瞰し、設計・開発・運用の品質を高める。
- 関係者間の認識合わせや説明資料の基礎とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- C4コンテキスト図（Mermaid記法等）
- 図の構成要素・関係性の説明
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- システム全体像を明確にすることで、設計・開発・運用の品質と効率を高める。
- 関係者間の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- システム全体像が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- 認識ズレによる品質低下・トラブルにつながる。

## サンプル

### C4コンテキスト図（小売業務システム例/Mermaid記法）

```mermaid
graph TD
 ((顧客)) -- 購入依頼 --> ((POSシステム))
 ((POSシステム)) -- 売上データ送信 --> ((販売管理システム))
 ((POSシステム)) -- 在庫照会 --> ((在庫管理システム))
 ((販売管理システム)) -- 売上集計 --> ((経営者))
 ((在庫管理システム)) -- 発注データ送信 --> ((仕入先システム))
```

---

注：上記は例です。実プロジェクトのC4コンテキスト図を具体的に記載してください。
