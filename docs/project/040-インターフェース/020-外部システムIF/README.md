# 020-外部システムIF

他システム（仕入先など）とのファイル連携やデータ受け渡し仕様を記述します。

## ドキュメントの目的

- 他システム（仕入先・会計・物流等）との連携仕様（ファイル形式、データ項目、連携タイミング、エラー処理等）を明確化し、設計・開発・運用の品質を高める。
- 連携設計・運用・障害対応の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- 外部システムIF仕様一覧表（連携先、連携内容、ファイル形式、項目一覧、連携タイミング、エラー処理等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- 外部連携仕様を明確にすることで、設計・開発・運用の品質と効率を高める。
- 連携設計・運用・障害対応の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- 外部連携仕様が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- 連携設計・運用・障害対応の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### 外部システムIF仕様（小売業務システム例/表形式）

| 連携先           | 連携内容         | ファイル形式 | 項目一覧                       | 連携タイミング     | エラー処理・備考         |
|------------------|------------------|-------------|------------------------------|-------------------|-------------------------|
| 仕入先システム    | 発注データ送信   | CSV         | 発注ID、発注日、商品コード、数量 | 発注確定時         | 異常時は再送・管理者通知 |
| 会計システム      | 売上データ送信   | CSV         | 販売ID、販売日、金額、顧客ID     | 日次集計後         | 異常時はエラーログ記録   |
| 物流システム      | 入荷データ受信   | CSV         | 入荷ID、入荷日、商品コード、数量 | 入荷完了時         | 異常時は管理者通知       |

---

注：上記は例です。実プロジェクトの外部システムIF仕様を具体的に記載してください。
