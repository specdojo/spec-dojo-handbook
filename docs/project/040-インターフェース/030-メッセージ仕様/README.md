# 030-メッセージ仕様

内部イベントやメッセージキューのメッセージ形式を記述します。

## ドキュメントの目的

- システム内部イベント・メッセージキューのメッセージ仕様（イベント名、項目一覧、フォーマット、送信元・送信先、エラー処理等）を明確化し、設計・開発・運用の品質を高める。
- メッセージ設計・実装・運用・障害対応の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- メッセージ仕様一覧表（イベント名、項目一覧、フォーマット、送信元・送信先、エラー処理等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- メッセージ仕様を明確にすることで、設計・開発・運用の品質と効率を高める。
- メッセージ設計・実装・運用・障害対応の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- メッセージ仕様が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- メッセージ設計・実装・運用・障害対応の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### メッセージ仕様（小売業務システム例/表形式）

| イベント名         | 項目一覧                       | フォーマット | 送信元         | 送信先         | エラー処理・備考         |
|--------------------|-------------------------------|-------------|---------------|---------------|-------------------------|
| 販売完了           | 販売ID、販売日、商品コード、数量 | JSON        | POS           | 販売管理      | 異常時は再送・管理者通知 |
| 在庫更新           | 商品コード、在庫数、更新日時     | JSON        | 販売管理      | 在庫管理      | 異常時はエラーログ記録   |
| 発注生成           | 発注ID、発注日、商品コード、数量 | JSON        | 在庫管理      | 発注管理      | 異常時は管理者通知       |

---

注：上記は例です。実プロジェクトのメッセージ仕様を具体的に記載してください。
