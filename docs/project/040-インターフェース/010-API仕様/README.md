# 010-API仕様

画面や外部から利用されるREST APIの仕様をOpenAPI形式で管理します。

## ドキュメントの目的

- 画面・外部システムから利用されるAPIの仕様（エンドポイント、リクエスト・レスポンス、認証、エラー等）を明確化し、設計・開発・運用の品質を高める。
- API設計・実装・テスト・連携の根拠とする。

## ドキュメントの内容

ドキュメントには、少なくとも以下を含めます。

- API仕様一覧表（エンドポイント、メソッド、概要、リクエスト・レスポンス例、認証等）
- サンプル（小売業務システム例）

## なぜこのドキュメントが必要か

- API仕様を明確にすることで、設計・開発・運用の品質と効率を高める。
- API設計・実装・テスト・連携の認識ズレ・トラブルを防ぐ。

## このドキュメントがないとどう困るか

- API仕様が曖昧になり、設計・開発・運用で誤解や手戻りが発生する。
- API設計・実装・テスト・連携の認識ズレによる品質低下・トラブルにつながる。

## サンプル

### API仕様（小売業務システム例/表形式）

| エンドポイント      | メソッド | 概要             | リクエスト例                | レスポンス例                | 認証 |
|---------------------|---------|------------------|----------------------------|-----------------------------|------|
| /api/inventory      | GET     | 在庫一覧取得      | なし                       | 商品コード、商品名、在庫数… | 必須 |
| /api/order         | POST    | 発注登録          | 商品コード、数量            | 発注ID、発注日              | 必須 |
| /api/sales         | POST    | 販売登録          | 商品コード、数量、顧客ID     | 販売ID、販売日              | 必須 |

---

注：上記は例です。実プロジェクトのAPI仕様を具体的に記載してください。
