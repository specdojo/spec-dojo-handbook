# 非機能要件 / 操作性 作成の指示テンプレート

以下のルールに従って、**非機能要件: 操作性（`nfr-usability`）** を 1 ファイル作成してください。出力は **Markdown** とします。

- 操作性は使いやすさ・誤操作防止・学習容易性を基準化します。
- UI詳細設計ではなく、利用結果としての品質基準を記述してください。

## 事前に与えられる情報（入力）

- 対象画面、対象業務、利用者属性（熟練度）
- 操作性指標（タスク完了率、平均完了時間、誤操作率）
- 画面仕様（`uis-*`）と検証先（`sac-*` / `sts-*`）

## 出力フォーマット（必須）

### YAML Frontmatter（必須）

```yaml
---
id: nfr-usability
type: architecture
title: 非機能要件: 操作性
status: draft # draft / ready / deprecated
part_of: [nfr-index]
based_on: []
supersedes: []
---
```

### 本文の見出し（必須・順序固定）

1. 概要（対象・目的）
2. 適用範囲・利用者前提
3. 操作性要件一覧
4. 測定・検証方法
5. 関連ドキュメント導線

## 記述ルール（重要）

- 要件一覧は表形式（`id` / `内容` / `指標` / `基準` / `備考`）で記載する。
- 指標はタスク完了率、平均完了時間、誤操作率、習熟時間などを使用する。
- 測定・検証方法はユーザ検証、受入試験、ログ分析で判定可能にする。

## 禁止事項

- 「使いやすい」だけで指標を定義しない。
- 画面詳細仕様を丸ごと再掲する。
- 利用者前提を書かない。

## 最終チェック（自己検査）

- `id: nfr-usability` / `type` / `status` が設定されている
- 見出しが指定順序で揃っている
- 適用範囲に利用者前提がある
- 要件一覧に `指標` / `基準` がある
- 測定・検証方法に判定タイミングと判定者がある
