## 記述ルール（重要）

- 要件一覧は表形式（`id` / `内容` / `指標` / `基準` / `備考`）で記載する。
- 指標は認可逸脱件数、脆弱性修正SLA、監査ログ欠落率などを使用する。
- 測定・検証方法はセキュリティ試験、監査ログ検証、スキャンで判定可能にする。

## 禁止事項

- 「安全にする」だけで基準がない記述。
- 認可・監査の要件を曖昧化する。
- 例外運用を本文に散在させる。
