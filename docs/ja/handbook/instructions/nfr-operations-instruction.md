# 非機能要件 / 運用 作成の指示テンプレート

以下のルールに従って、**非機能要件: 運用（`nfr-operations`）** を 1 ファイル作成してください。出力は **Markdown** とします。

- 運用は監視可能性、通知、手動介入条件、復旧性を基準化します。
- 手順そのものではなく、満たすべき運用結果を記述してください。

## 事前に与えられる情報（入力）

- 対象システム、監視時間、オンコール体制
- 通知経路、運用指標（MTTD、誤検知率、バックアップ成功率）
- 運用設計/手順（`opd-*` / `opr-*`）と受入条件（`sac-*`）

## 出力フォーマット（必須）

### YAML Frontmatter（必須）

```yaml
---
id: nfr-operations
type: architecture
title: 非機能要件: 運用
status: draft # draft / ready / deprecated
part_of: [nfr-index]
based_on: []
supersedes: []
---
```

### 本文の見出し（必須・順序固定）

1. 概要（対象・目的）
2. 適用範囲・体制前提
3. 運用要件一覧
4. 測定・検証方法
5. 関連ドキュメント導線

## 記述ルール（重要）

- 要件一覧は表形式（`id` / `内容` / `指標` / `基準` / `備考`）で記載する。
- 指標は MTTD、通知到達時間、バックアップ成功率などを使用する。
- 測定・検証方法は監視ダッシュボード、運用レポート、訓練結果で判定可能にする。

## 禁止事項

- 手順書全文を再掲する。
- 通知先や判定条件を曖昧にする。
- 監視指標を定義しない。

## 最終チェック（自己検査）

- `id: nfr-operations` / `type` / `status` が設定されている
- 見出しが指定順序で揃っている
- 要件一覧に `指標` / `基準` がある
- 測定・検証方法に判定タイミングと判定者がある
- 関連ドキュメント導線が空欄でない
