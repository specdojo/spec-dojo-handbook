# 非機能要件 / 完全性 作成の指示テンプレート

以下のルールに従って、**非機能要件: 完全性（`nfr-integrity`）** を 1 ファイル作成してください。出力は **Markdown** とします。

- 完全性は正確性・整合性・改ざん耐性を基準化します。
- 参照整合、重複防止、監査追跡性を検証可能な条件で記述してください。

## 事前に与えられる情報（入力）

- 対象データ、整合性境界、監査要件
- 改ざん防止要件、履歴保持方針、例外条件
- 検証先（`sac-*` / `sts-*`）と判断記録（`adr-*`）

## 出力フォーマット（必須）

### YAML Frontmatter（必須）

```yaml
---
id: nfr-integrity
type: architecture
title: 非機能要件: 完全性
status: draft # draft / ready / deprecated
part_of: [nfr-index]
based_on: []
supersedes: []
---
```

### 本文の見出し（必須・順序固定）

1. 概要（対象・目的）
2. 適用範囲・前提条件
3. 完全性要件一覧
4. 測定・検証方法
5. 関連ドキュメント導線

## 記述ルール（重要）

- 要件一覧は表形式（`id` / `内容` / `指標` / `基準` / `備考`）で記載する。
- 指標は整合性違反件数、監査証跡欠落率、重複発生件数などを使用する。
- 測定・検証方法は整合性チェック、監査レビュー、試験で追える形にする。

## 禁止事項

- 整合性違反を「原則なし」で済ませる。
- 改ざん防止要件に検証方法がない。
- 監査証跡の保持基準を省略する。

## 最終チェック（自己検査）

- `id: nfr-integrity` / `type` / `status` が設定されている
- 見出しが指定順序で揃っている
- 要件一覧に `指標` / `基準` がある
- 測定・検証方法に判定タイミングと判定者がある
- 関連ドキュメント導線が空欄でない
